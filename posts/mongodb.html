MongoDB Database & MongoDB Atlas

EL H.TOUZA

MongoDB is a NoSQL database that allows us to store the data ( the content of our website or web application ) in collections of documents that are very similar to JavaScript objects. MongoDB database used with other backend technologies like Node JS, Express, React or Angular to build a full stack web applications.

1 - What is NoSQL Database ?

NoSQL stands for "Not Only SQL", that means we could use SQL on some of these databases, but a better word to describe a NoSQL database is a Non-relational database, so there is no tables, rows or culomns like we use to have in a SQL database, instead we use collections of documents to store the data in NoSQL database.

2 - MongoDB Basics

MongoDB is Built on top of two core concepts that are documents and collections, in these section we will take a look at these two core concepts and how they work on MongoDB.

documents :

MongoDB is a document-oriented database, which means that the equivalent of a record ( composed fields contain all the data about one particular item, user or product in a database ) is a document, or object. in a SQL database (relational database) we store data in terms of rows and culomns, whereas is document database like MongoDB the entire object that contains all data about a specific item can be written as a document.

A document is data structure, a way to store data in the database composed of fields and value pairs and it can contain arrays and other objects in a nested manner. MongoDB document has support not only for primitive data types - Booleans, numbers an strings - but also other comon data types such as dates, timestamps, regular expressions, and binary data.
